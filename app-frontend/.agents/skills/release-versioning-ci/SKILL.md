## Activation Contract

Use when changing release workflow, version metadata, GitHub Actions, signing, or distribution.

## Hard Rules

- Treat `app.version` in `gradle.properties` as generated by semantic-release/version sync, not hand-edited casually.
- Keep `scripts/sync-version.mjs` as the single version propagation path.
- Keep alpha/prod release workflows branch-specific.
- Do not weaken signing, upload, or distribution secrets handling.
- Never publish or push unless the user explicitly asks.

## Decision Gates

| Concern                      | Owner                      |
| ---------------------------- | -------------------------- |
| Semantic version calculation | `.releaserc.yml`           |
| Gradle/iOS metadata sync     | `scripts/sync-version.mjs` |
| Android distribution         | Android composite action   |
| Desktop artifacts            | desktop composite action   |
| Artifact signing/upload      | sign-blob action           |

## Execution Steps

1. Identify release lane: alpha or prod.
2. Update version sync/release config consistently.
3. Validate scripts without exposing secrets.
4. Report artifact and secret assumptions.

## Output Contract

Report release lane, metadata files, CI actions, and validation performed.
